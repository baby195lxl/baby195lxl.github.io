Vim中的几个易混淆概念

Buffer：你打开的文件本体

很多人以为打开文件就是打开一个窗口，但在 Vim/Neovim 里，Buffer 才是真正的文件——它是文件加载到内存中的副本，是所有操作的基础。
举个类比：你在电脑里打开了一个 Word 文档，这个文档在内存中对应的临时副本，就是 Vim 里的 Buffer。哪怕你关掉了显示这个文档的窗口，只要没手动关闭 Buffer，它就还在内存里，随时可以重新调出来。
关键特点：

  • 一个文件对应一个 Buffer，打开多个文件，就会创建多个 Buffer；
  • Buffer 是看不见摸不着的，只能通过 Window 来显示它；
  • 关闭 Window 不会关闭 Buffer，只有执行关闭 Buffer 的命令，文件才会从内存中释放。

Window：查看 Buffer 的视口

如果说 Buffer 是文件本体，那 Window 就是用来查看这个文件的窗口——它只是一个显示工具，本身不存储文件内容，只负责展示某个 Buffer 的内容。
类比一下：Buffer 是你手机里的照片，Window 就是你手机的屏幕。你可以用一个屏幕(Window)看一张照片(Buffer)，也可以把屏幕分屏(多个 Window)，同时看两张不同的照片(不同 Buffer)，甚至用两个屏幕看同一张照片(两个 Window 显示同一个 Buffer)。
关键特点：

  • Window 是可视化的容器，没有 Buffer，Window 就是空的；
  • 一个 Window 只能显示一个 Buffer，但一个 Buffer 可以被多个 Window 显示；
  • 切换 Window 只是切换“当前查看的视口”，不会改变 Buffer 的状态。

Tab：Window 布局的保存器

这是最容易和其他编辑器混淆的概念——Vim/Neovim 的 Tab 不是文件标签！而是一套 Window 布局的容器。
很多人习惯了 VSCode 的 Tab：一个 Tab 对应一个文件。但在 Vim 里，Tab 更像是工作区——你可以把一套分屏布局(比如左边一个 Window 看代码，右边一个 Window 看文档)保存到一个 Tab 里，切换 Tab 就相当于切换一整套布局。
类比一下：你在电脑上打开了两个工作区，第一个工作区是左边浏览器、右边笔记，第二个工作区是全屏编辑器。Vim 的 Tab 就相当于这两个工作区，切换 Tab 就是切换整个工作区的布局，而不是切换单个文件。
关键特点：

  • 一个 Tab 可以包含多个 Window，这些 Window 组成一套固定布局；
  • 切换 Tab 时，会完整加载该 Tab 保存的所有 Window 及其对应的 Buffer；
  • 关闭 Tab 会关闭该 Tab 下的所有 Window，但不会关闭对应的 Buffer(Buffer 仍在内存中)。
